# Automated Data Ingestion & Pipeline Integration Plan

This plan details the addition of automated data collection scripts (Task 1 & 2) and their integration into the existing RAG pipeline (Task 3) while adhering strictly to the non-destructive constraints and established architectural patterns.

## User Review Required

Please review the proposed architectural changes below, especially how the scraping scripts will be integrated into the existing `ingest.py` CLI. 

> [!IMPORTANT]
> The Physiopedia URL provided is a Category page (`Category:Therapeutic_Exercise`), which primarily contains links to individual exercise articles rather than full content itself. 
> **Open Question:** Should the scraper strictly extract text from the Category page itself, or should it fetch the Category page, extract the article links, and scrape the individual article pages (with a `time.sleep(2)` delay between each)? The plan assumes we will scrape the individual articles linked from the category page to get meaningful text.

> [!NOTE]
> The APTA Orthopedics CPGs URL is an index page. The script will use BeautifulSoup to find all `.pdf` links on this index page and download them.

### Scrapers and Downloaders (New Files)

I will create two new robust, error-handled python scripts in a new `src/scrapers/` directory.

#### [NEW] [physiopedia_scraper.py](file:///c:/Users/james/OPIM-5517-Project/src/scrapers/physiopedia_scraper.py)
- Uses `requests` and `BeautifulSoup`.
- Fetches the `Category:Therapeutic_Exercise` page, extracts article links, and visits them.
- Extracts ONLY text within `<h1>`, `<h2>`, and `<p>` tags (skipping nav, sidebars, and menus).
- Implements ethical scraping: standard User-Agent headers, `time.sleep(2)` delays, and `try/except` for network resilience.
- Saves formatted text directly into `data/pt/unstructured/`.

#### [NEW] [clinical_downloader.py](file:///c:/Users/james/OPIM-5517-Project/src/scrapers/clinical_downloader.py)
- Uses `requests` and `BeautifulSoup`.
- Fetches the APTA CPGs index page, finds all `.pdf` hrefs, and downloads them.
- Fetches the Evidence-Based Massage Therapy OER textbook directly from the provided PDF URL.
- Saves all downloaded PDFs directly into `data/pt/structured/`.
